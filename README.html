Colossus Cognitive Environment (CCE)

CCE is a private, bounded operational substrate for a Matrix/Element-to-Colossus channel. Its purpose is not to automate broadly; its purpose is to reduce cognitive friction with explicit interfaces, inspectable state, and verifiable completion.

This repository does not define a general autonomous agent. Phase 1 established the foundation; Phase 2 adds the smallest possible verified Matrix round-trip runtime through a narrow gateway boundary.

Current architecture

  • matrix/: communication-layer boundary for Matrix/Element integration
  • models/: local reasoning-layer invocation boundary
  • tools/: deterministic bounded operational tools
  • pem/: epistemic governance and continuity integration boundary
  • workspaces/: durable bounded truth containers defined by manifests
  • gateway/: narrow orchestration layer tying the boundaries together
  • cases/: PEM-backed continuity and forensic case records
  • scratch/: non-authoritative ephemeral working area
  • pem.initialized: explicit PEM initialization marker
  • docs/: explicit architecture, scope, contracts, and decisions

Hermes is treated as a prototype and lesson source, not as the foundation of CCE.

What CCE is not

  • Not a broad autonomous agent
  • Not a hidden-memory system
  • Not a vector-database or embeddings project
  • Not a broad shell or filesystem access layer
  • Not a Docker orchestration expansion
  • Not a multi-agent framework

Phase 2 runtime

Phase 2 provides a minimal Matrix listener/replier with one safe command:

  • status

The runtime:

  • listens in one configured Matrix room
  • ignores its own messages
  • replies to status with a verified state snapshot
  • replies to unknown commands with Unknown command. Supported commands: status

The runtime does not add model invocation, PEM calls, document reading, shell execution, broad filesystem access, or hidden memory.

Phase 3 workspace runtime

Phase 3 adds a deterministic workspace runtime with explicit manifests and explicit sources.

Gateway commands

  • workspace list
  • workspace open <id>
  • workspace status
  • sources
  • read <source>

Only .md and .txt sources are readable, and only if explicitly declared in the active workspace manifest.

Phase 3.1 persists the active workspace selection across manual gateway restarts in runtime/workspace-state.json (gitignored, human-inspectable operational state).

Phase 4 local model boundary

Phase 4 adds a bounded ask <question> command. The model sees only declared readable sources from the active workspace (.md/.txt), assembled deterministically with explicit byte limits and a timeout.

This is not web browsing, retrieval, memory, or autonomous behavior.

Phase 4.1 routes ordinary non-command questions to the same grounded QA path when an active workspace is open.

Phase 4.2 hardens Matrix auth failures; see docs/matrix-auth-hardening.md.

Phase evaluation

Current repository evaluation should check whether it provides:

  1. A clear, boring project structure with named operational boundaries
  2. Explicit written contracts for gateway, workspace manifests, PEM role, and completion semantics
  3. A narrow runtime that proves Matrix ingress and egress without broadening scope
  4. Explicit documentation of first-class state surfaces, including case records and PEM initialization state
  5. A documented list of non-goals and deferred work
  6. A decision log and target document that make tradeoffs inspectable

See docs/phase1-scope.md, docs/architecture.md, docs/state-surfaces.md, and docs/june18-target.md for the authoritative Phase 1 foundation.

Configuration

Create config/cce.json from config/cce.example.json and provide:

  • homeserver_url
  • access_token
  • user_id
  • room_id

Real credentials are ignored by git.

Run

Config check

python3 -m gateway.runtime --config config/cce.json --check-config

Matrix connectivity check

python3 -m gateway.runtime --config config/cce.json --check-connection

Optional: override Matrix cursor state path

python3 -m gateway.runtime --config config/cce.json --state-path runtime/matrix-sync-state.json

Listener runtime

python3 -m gateway.runtime --config config/cce.json

Test

python3 -m unittest tests.test_gateway_status
