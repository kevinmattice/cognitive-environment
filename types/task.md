# Task

A Task is a specific unit of work to do.

## When to use it

Use this type for concrete actions that can be completed, deferred, or tracked.

## Examples

- A future task to add more entity notes

## Not to be confused with

Do not confuse a Task with the broader project it belongs to or with a decision that authorizes it.
