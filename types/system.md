# System

A System is a named functional whole with behavior, boundaries, and recognizable responsibilities.

## When to use it

Use this type when something operates as a coherent unit rather than a loose idea or isolated file.

## Examples

- [[PEM]]
- [[Matrix]]

## Not to be confused with

Do not confuse a System with a hosting substrate like [[Colossus]], or with a single capability such as [[Memory]].
